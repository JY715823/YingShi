# Current Task: Stage 12.3 - 上传 / 回收站 / 媒体管理压测修复

## 背景

Stage 12.2 已完成主链路状态一致性收口。现在进入 Stage 12.3，重点压稳复杂媒体操作链路，包括上传、系统媒体导入、加入已有帖子、发成新帖子、Gear Edit 媒体管理、目录删、系统删、空帖保护、回收站恢复和 24h 撤销入口。

本阶段不是新增大功能，而是对已有功能做压测修复和失败兜底。

## 目标

确保以下链路在 FAKE / REAL 模式下都稳定：

1. 系统媒体发成新帖子
2. 系统媒体加入已有帖子
3. Gear Edit 媒体管理
4. 上传成功 / 失败
5. 目录删
6. 系统删
7. 空帖保护
8. 回收站恢复
9. 24h 撤销入口

## 范围

### 1. 上传成功 / 失败

需要保证：

- 上传成功后照片流刷新
- 上传成功后帖子详情媒体区刷新
- 上传成功后 Gear Edit 媒体管理刷新
- 上传失败不把假数据塞进列表
- 上传中有状态
- 上传失败有中文提示或重试
- 多个媒体连续上传不串状态

### 2. 系统媒体发成新帖子

需要保证：

- 从系统媒体选择媒体后可发成新帖子
- 成功后相册 / 帖子列表刷新
- 新帖子媒体关系正确
- 上传或创建失败时不产生半成品 UI 状态
- FAKE / REAL 行为一致

### 3. 系统媒体加入已有帖子

需要保证：

- 二级选择器稳定：先选相册，再选帖子
- 加入成功后帖子详情和 Gear Edit 媒体管理刷新
- 加入失败不污染帖子媒体列表
- 重复加入时有合理提示或安全处理
- 当前帖子媒体排序不乱

### 4. Gear Edit 媒体管理

需要检查：

- 设封面
- 排序
- 删除模式
- 目录删
- 系统删
- 多媒体连续操作
- 删除后刷新
- 操作失败兜底

### 5. 目录删 / 系统删

目录删要求：

- 只移除当前帖子和媒体关系
- 不影响照片流全局媒体
- 不进入系统媒体回收站
- 按现有业务进入 app 回收站或记录对应删除态

系统删要求：

- 影响全局媒体
- 从所有帖子关系中移除
- 进入 app 回收站
- 照片流、帖子详情、Gear Edit、Viewer 都不继续显示旧媒体

### 6. 空帖保护

需要保证：

- 删除最后一张媒体前有保护
- 不允许把帖子删成非法空帖，除非现有业务明确允许
- 提示文案自然
- 删除失败或取消时状态不变
- Viewer 正在查看最后一张媒体时不闪退

### 7. 回收站恢复与 24h 撤销

需要保证：

- 恢复帖子后相册列表重新出现
- 恢复媒体后照片流重新出现
- 恢复后回收站列表移除
- 24h 撤销入口可找到
- 撤销成功后相关列表刷新
- 不再长期用底部提示打扰用户

### 8. 系统媒体工具区

需要保证：

- 发成新帖子后刷新
- 加入已有帖子后刷新
- 移到系统回收站后刷新
- 用户取消系统确认时不改状态
- 权限不支持时中文提示，不闪退

## Stage 12.3 实际收口补充

- 系统媒体“发成新帖子 / 加入已有帖子”改为“入队成功”和“完成成功”分离：入队只提示已加入上传队列，真正成功后再通过统一完成事件刷新照片流、帖子详情、Gear Edit 媒体管理，并在新帖创建成功后打开新帖子。
- REAL 上传如果文件已经落到后端但还没有成功挂到任一帖子，不再出现在主照片流，避免上传失败或整理帖子失败时污染列表。
- REAL 删除链路补了空帖保护：删除最后一张媒体会被后端拒绝，前端保留中文提示，避免目录删 / 系统删把帖子留成空帖。
- 回收站、24h 撤销、系统媒体工具区仍沿用 Stage 12.2 的 mutation bus 刷新策略，这一轮只补复杂上传和删除链路的最终一致性。
