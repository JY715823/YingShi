# Roadmap

> 当前路线图按“先壳层、再模块、后能力接入”的顺序推进。  
> 现阶段以本地假数据、占位页面、清晰文档边界为先。

## Stage 0

仓库初始化与协作基线。

目标：

- 补齐 README、AGENTS、roadmap、current-task
- 建立可运行的 Android UI 壳层
- 建立全局底部导航占位
- 建立照片模块顶部二级导航占位
- 建立 feature-oriented 基础目录

不做：

- 真实后端
- 真实评论
- 真实 Viewer
- 真实系统媒体

## Stage 1

设计系统与全局壳层稳定化。

- theme token 整理
- 通用页面骨架
- 更清晰的导航样式
- 空态、占位态、基础组件约定

## Stage 2

照片页 v1，先做“全局媒体流”的本地假数据版本。

- 假数据模型
- 网格布局
- 时间分组
- 系统媒体入口占位
- 通知入口占位

## Stage 3

Viewer v1，占位逻辑到基础交互壳层。

- 全屏查看态结构
- 左右切换骨架
- 缩放交互占位
- 边缘操作层占位
- 评论预览层壳子

## Stage 4

相册页 + 帖子详情页 v1。

- 相册列表
- 帖子卡片
- 帖子详情信息白区
- 帖子评论区占位

## Stage 5

评论系统 v1。

- 帖子评论分流
- 媒体评论分流
- 评论输入与列表
- 评论加载策略

## Stage 6

Gear Edit 与帖子媒体管理。

- 帖子编辑页
- 媒体管理页
- 排序
- 封面设定
- 帖子内删除语义分叉

## Stage 7

删除与回收站。

- 帖子删除
- 目录删 / 系统删
- 回收站分段
- 删除态详情与恢复

## Stage 8

系统媒体工具区。

- 系统媒体网格
- 多选整理
- 发成帖子
- 加入已有帖子
- 移到系统回收站

## Stage 9

原图、长图、视频、缓存。

- 原图按需加载
- 全帖原图加载
- 长图处理
- 视频预览 / 播放
- 缓存策略

## Stage 10

通知中心与设置。

- 通知分类
- 设置页
- 权限引导
- 缓存管理
- 浏览偏好

## Stage 11

后端契约与真实接入。

- API 契约
- 仓库层替换
- 上传下载链路
- 真实同步

## Stage 12

精修与稳定性。

- 状态收口
- 性能优化
- 边界场景
- UI 精修
- 预发布检查
