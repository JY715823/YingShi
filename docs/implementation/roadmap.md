# Roadmap

> 当前路线按“先壳层、再模块、后真实能力接入”的顺序推进。  
> 现阶段优先本地假数据、占位页面和清晰文档边界。

## Stage 0

仓库初始化与协作基线。

- 补齐 `README.md`、`AGENTS.md`、`roadmap.md`、`current-task.md`
- 建立最小可运行 Android app 壳层
- 建立底部一级导航与照片模块顶部二级导航
- 保持仓库结构适合阶段化 AI 协作

不做：

- 真实后端
- 真实评论
- 真实 Viewer
- 真实系统媒体

## Stage 1

设计系统与全局壳层稳定化。

- theme token 整理
- 页面骨架与导航风格稳定
- 底栏与照片模块顶部结构定稿
- 占位页、空态、基础组件基线

## Stage 2

照片页 v1，先做“全局媒体流”的本地假数据版本。

- 假数据模型与假 repository
- 媒体去重与时间降序
- 网格密度切换：`2 / 3 / 4 / 8 / 16`
- 随密度变化的时间分组逻辑
- 纯缩略图媒体卡
- 长按占位与后续多选挂点

## Stage 3

Viewer v1，从占位走到基础交互壳层。

- 全屏查看态结构
- 左右切换骨架
- 缩放交互占位
- 边缘操作层占位
- 评论预览层壳子

## Stage 4

相册页与帖子详情页 v1。

- 相册列表
- 帖子卡片
- 帖子详情信息白区
- 帖子评论区占位

## Stage 5

评论系统 v1。

- 帖子评论分流
- 媒体评论分流
- 评论输入与列表
- 评论加载策略

## Stage 6

Gear Edit 与帖子媒体管理。

- 帖子编辑页
- 媒体管理页
- 排序
- 封面设定
- 帖子内删除语义分叉

## Stage 7

删除与回收站。

- 帖子删除
- 目录删 / 系统删
- 回收站分段
- 删除态详情与恢复

## Stage 8

系统媒体工具区。

- 系统媒体网格
- 多选整理
- 发成帖子
- 加入已有帖子
- 移到系统回收站

## Stage 9

原图、长图、视频与缓存。

- 原图按需加载
- 全帖原图加载
- 长图处理
- 视频预览 / 播放
- 缓存策略

## Stage 10

通知中心与设置。

- 通知分类
- 设置页
- 权限引导
- 缓存管理
- 浏览偏好

## Stage 11

后端契约与真实接入。

- API 契约
- 仓库层替换
- 上传下载链路
- 真实同步

## Stage 12

精修与稳定性。

- 状态收口
- 性能优化
- 边界场景
- UI 精修
- 预发布检查
